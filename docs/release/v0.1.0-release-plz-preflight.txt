Common failure surface:

- `release-plz` reports `failed to retrieve difference of package fret-a11y-accesskit`.
- The nested cause is `can't checkout to head after calculating diff`.
- The underlying git error is:
  - `error while running git ... ["checkout", "main"]`
  - `Your local changes to the following files would be overwritten by checkout:`
  - `ecosystem/fret-ui-shadcn/src/slider.rs`

Root cause:

- The current repository working tree did not have `ecosystem/fret-ui-shadcn/src/slider.rs` modified when this was tested.
- The checkout conflict therefore appears inside the temporary clone/worktree that `release-plz` uses while computing package differences.
- `git_cmd` checks out entire historical commits while computing package diffs, not only the package path under inspection.
- Several historical shadcn source blobs carry CRLF content even though the repo enforces `*.rs text eol=lf`, so those files appear as EOL-only dirty changes during checkout recovery.
- Confirmed historical offenders:
  - `ecosystem/fret-ui-shadcn/src/slider.rs` at `ede40b3c88bfc34f86fe4bb4021e74cebf095017`
  - `ecosystem/fret-ui-shadcn/src/context_menu.rs` at `d027f2070ce3e50c86a659180b9be434aa6b06a7`
  - `ecosystem/fret-ui-shadcn/src/dropdown_menu.rs` at `d027f2070ce3e50c86a659180b9be434aa6b06a7`
  - `ecosystem/fret-ui-shadcn/src/menubar.rs` at `d027f2070ce3e50c86a659180b9be434aa6b06a7`
  - `ecosystem/fret-ui-shadcn/src/tabs.rs` at `7c9029de7a97b2af1b4fbcbb5c80c725a3e24b6d`
  - `docs/adr/IMPLEMENTATION_ALIGNMENT.md` at `3111664eb32312fd63e38cc60b6f454e1f4b8a7a`

Current operator guidance:

- Treat `release_closure_check.py` plus wave-specific `cargo publish --dry-run` as the authoritative local publish-readiness gates for now.
- Run `python3 tools/prepare_release_plz_checkout_overrides.py` before local `release-plz update` commands.
- `release-plz update` is now validated at package scope (`fret-assets`, `fret-a11y-accesskit`, `fret-app`, `fret-render`) and at full-workspace scope, provided the managed checkout override block is installed first.
- Do not treat the raw historical `release-plz update` failure surface as evidence that the release scope or publish order is wrong; the blocker was the history checkout recovery path for CRLF-bearing blobs.
